From Legacy Monolith to Multi-Tenant Healthcare Platform: A Digital Transformation Story

Company Periscope Technologies / DexCare
Role Technical Project Manager
Duration Jul 2025 – Mar 2026
Team Size 25 Members
70+Microservices Managed
Multi-tenantArchitecture Delivered
DatabricksUnified Data Platform
AI-FirstDelivery Model

The Challenge

DexCare's healthcare data platform was built on a single-tenant, monolithic architecture ingesting provider data from multiple source systems — Epic, Salesforce, Kyruus, and SFTP feeds — through fragmented, client-specific pipelines. As the client base grew, the system became a bottleneck: each new onboarding required custom engineering, data inconsistencies proliferated, and the legacy Omni services couldn't scale to meet search performance demands.

The mandate: transform a legacy, inflexible platform into a shared, multi-tenant architecture that could handle 70+ microservices, standardize data ingestion across all clients, and dramatically reduce onboarding time — without taking the platform offline.

My Role

Technical Project Manager — end-to-end program ownership across distributed engineering, data, DevOps, and QA teams. Primary client SPOC. Responsible for delivery governance, cross-team dependency management, executive reporting, and Agile facilitation across all workstreams.

The Approach

Architecture Transformation

  • Drove transition from fragmented, client-specific Airflow ingestion jobs to a unified Databricks-based ingestion DAG framework, standardizing data pipelines across all clients.
  • Managed migration from legacy Omni APIs to Core APIs with SPI as the central provider data store, decoupling ingestion, data management, and application layers.
  • Coordinated Auth0 + Kong Gateway + AWS EKS integration to enable secure multi-tenancy, OAuth 2.0 token flows, and faster client onboarding.
  • Oversaw ActionIQ change request management module delivery and enhancements across multiple PDM modules in the Feb 2026 major release milestone.

Delivery Governance

  • Ran Agile/Kanban ceremonies across a 25-member cross-functional team spanning frontend (React 19, TypeScript, MUI), backend (microservices), data engineering (Databricks, Step Functions), and DevOps (Docker, Helm, CI/CD on AWS).
  • Built executive dashboards tracking cross-team API readiness, dependency resolution, and release health — enabling real-time visibility for senior stakeholders.
  • Managed risk proactively: API sequencing, shared service dependencies, and CI/CD pipeline stability were tracked as first-class delivery risks with mitigation plans updated weekly.
  • Established code freeze protocols, QA sign-off gates, and release readiness reviews to ensure stable production deployments.

AI-Augmented Delivery

  • Leveraged Claude, Perplexity AI, and GitHub Copilot to accelerate requirements analysis, ticket drafting, and QA validation documentation — a first for the delivery team.
  • Embedded AI tooling into sprint workflows, reducing documentation overhead by approximately 30–40% and freeing engineering capacity for feature delivery.
  • Used AI-assisted analysis to surface dependency conflicts in backlog before sprint planning, reducing mid-sprint blockers.

"The real challenge wasn't just the technology — it was coordinating 70+ microservices, multiple engineering squads, and a client with zero tolerance for downtime. The answer was ruthless prioritization, proactive dependency mapping, and building dashboards that told the truth."

Impact

  • Enabled migration from single-tenant to scalable multi-tenant architecture, significantly reducing infrastructure overhead per client and accelerating new client onboarding.
  • Databricks-based unified ingestion framework improved data standardization, reduced pipeline duplication, and improved API-driven accessibility across all client systems.
  • AI-augmented delivery workflows improved release documentation quality and sprint throughput across the team.
  • Successfully delivered the February 2026 major release milestone on schedule, including the new ActionIQ module, data grid performance improvements, Auth0 security integration, and significant DevOps/QA enhancements.

Key Learnings

  • Platform modernization at this scale requires a dedicated Technical PM who bridges engineering jargon and business outcomes — not just a scrum master facilitating standups.
  • AI tooling in delivery workflows is no longer optional; teams that integrate it cut documentation cycles by 30–40% and elevate the quality of stakeholder communication.
  • Multi-tenancy transitions need architectural dependency mapping before the first sprint begins — surprises at the API layer are the #1 delivery risk in platform transformation programs.
